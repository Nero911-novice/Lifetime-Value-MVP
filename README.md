
# LTV Demo MVP для райд-хейлинга

Демонстрационное аналитическое веб-приложение на Streamlit. Проект показывает, как может выглядеть будущий продукт для анализа LTV в райд-хейлинге на уровне MVP:

- управленческий обзор по базе пользователей;
- когортный анализ удержания и накопленного LTV;
- сегментация по риску, ценности и промо-зависимости;
- карточка пользователя;
- модель демонстрационных данных и словарь полей.

## Цель проекта

Это не продакшн-решение, а репрезентативный прототип для:
- публикации в GitHub;
- развертывания в Streamlit;
- презентации ЛПР;
- обсуждения состава данных, методики и следующего этапа разработки.

## Логика данных

В проекте используются четыре основные сущности:

1. `users` — пользовательский контекст, канал привлечения, тип активации, базовые признаки.
2. `trips` — поездки, GMV, выручка платформы, скидки, возвраты, contribution margin.
3. `marketing_touches` — история маркетинговых касаний и упрощенный response-подход.
4. `campaigns` — агрегированная информация о кампаниях.

Дополнительно есть:
- `data_dictionary.csv` — словарь полей;
- `dataset_summary.csv` — краткая сводка по таблицам.

## Что реализовано в MVP

### 1. Архитектура
- разделены интерфейс, данные, расчеты и тексты методологии;
- демонстрационные CSV можно заменить реальными витринами без полной переработки интерфейса.

### 2. Экраны
- **Обзор**
- **Когорты**
- **Сегменты**
- **Карточка пользователя**
- **Модель данных**

### 3. Методологическая прозрачность
- на каждом экране есть локальная справка;
- для ключевых показателей и сущностей добавлены пояснения;
- внизу каждого экрана размещена единая методологическая справка.

## Ограничения MVP

- расчеты основаны на синтетических демонстрационных данных;
- исторический LTV не заменяет прогнозный LTV;
- response по касаниям не является причинным uplift;
- сегменты реализованы по rule-based логике, а не через production-модели.

## Следующее развитие после MVP

1. Подключение к БД и замена CSV на витрины.
2. Отдельные витрины по событиям и календарным окнам.
3. Прогнозный LTV: count + survival + margin.
4. Реактивация и response-модели.
5. Uplift / incremental LTV и decision layer.
